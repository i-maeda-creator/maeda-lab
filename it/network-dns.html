Focus 01 / Network

DNS 名前を、通信できる住所へ変える仕組みです。

DNSは、order.maeda-lab.example のような人間向けの名前を、サーバーのIPアドレスへ変換する仕組みです。Web、メール、CDN、SaaS連携の入口で必ず働きます。

カフェのスタッフがブラウザに order.maeda-lab.example と入力します。人間には店名のように読めますが、ネットワーク機器は最終的にIPアドレスで相手を探します。DNSは、電話帳というより「分散された案内所」です。端末、社内DNS、権威DNS、CDNのDNSが順番に関わり、利用者を正しい入口へ案内します。

DNSがしていること

DNSの本質は「名前から必要な情報を引く」ことです。WebならA/AAAAレコードでIPアドレスを調べ、メールならMXレコードで配送先を調べ、送信ドメイン認証ならTXTレコードを見ます。つまりDNSは、Webだけの仕組みではなく、インターネット上のサービス設計そのものを支える台帳です。

レコードの意味

レコード役割具体例
A / AAAA名前をIPv4/IPv6へ対応させるorder.example -> 203.0.113.10
CNAME別名へ向けるwwwをCDNやSaaSのホスト名へ向ける
MXメールの配送先を示すMicrosoft 365やGoogle Workspace
TXT任意の文字情報を置くSPF、DKIM、DMARC、所有確認
NS権威DNSを示すドメインをどのDNS事業者が管理するか

実務でよく起きる事故

DNS移行で多いのは、TTLを下げずに切り替えて一部利用者だけ旧サーバへ行く事故です。もう一つは、Web用のCNAMEだけを見て、メール用のMX/TXTを壊してしまう事故です。DNSは小さな設定に見えますが、Web、メール、証明書、SaaS連携の交通整理をしているため、切替前には現在のレコードを棚卸しする必要があります。

Quick Quiz 1

名前をIPv4アドレスへ対応させる代表的なDNSレコードはどれでしょうか。

Aレコードは名前をIPv4アドレスへ対応させます。IPv6ならAAAAレコードです。

Applied Quiz 2

サーバ移行後、一部の利用者だけ旧IPへ接続します。まず疑うべきものはどれでしょうか。

古い答えが再帰DNSや端末に残る典型例です。TTL、再帰DNS、端末キャッシュを見ます。

Network Specialist Written

DNS 午後記述

問題の背景

カフェの注文サイトを旧サーバからCDN経由の新基盤へ移行しました。事前にAレコードを変更しましたが、移行後も一部の利用者は旧画面へ接続します。社内PCでは新画面が表示され、スマートフォンの一部では旧画面が残ります。DNS管理画面ではAレコードは新IPを指しています。さらに、apexドメインをCDNへ向けたいものの通常のCNAMEは設定できません。メールはMicrosoft 365を利用しており、SPF/DKIM/DMARCのTXTも同じゾーンにあります。

短答記述

一部利用者だけ旧IPへ接続する原因として、DNSの観点で考えられるものを三つ挙げ、それぞれ確認方法を説明してください。

事例ベースの午後問題

apexドメインをCDNへ向けつつ、メール認証レコードを壊さずに安全に切り替える手順を説明してください。TTL、再帰DNSキャッシュ、権威DNS、ALIAS/ANAME/CNAME flattening、MX/TXTレコードの保全、切戻し手順を含めて述べてください。

回答はこちらをクリック

模範解答・採点観点

  • 旧IPが残る理由として、TTLが長い、再帰DNSのキャッシュ、端末/ブラウザ/OSのキャッシュ、権威DNSの参照先違いを挙げる。
  • dig/nslookupで権威DNSと複数の再帰DNSへ問い合わせ、どこで旧IPが返っているか比較する。
  • apexドメインでは通常CNAMEを置けないため、DNS事業者のALIAS/ANAME/CNAME flattening機能を確認する。
  • MX/TXT/SPF/DKIM/DMARCを棚卸しし、Web切替でメール配送やドメイン認証を壊さないようにする。
  • 切替前にTTLを下げ、検証環境で確認し、段階切替、監視、切戻し手順を用意する。

解説

DNS障害で難しいのは、管理画面の表示が正しくても、利用者が見る答えがすぐには変わらないことです。DNSにはキャッシュがあります。再帰DNSや端末が古い答えを持っていれば、権威DNSを直しても一定時間は旧IPへ行くことがあります。だから、切替時は「権威DNSが正しいか」「よく使われる再帰DNSが何を返すか」「端末側で古い答えが残っていないか」を分けて確認します。

また、DNSはWebだけではありません。CDNへ向ける作業でMXやTXTを消すと、メールが届かない、SPF failになる、DMARCで拒否される、といった別事故が起きます。ネットワークスペシャリストの午後では、単にAレコードを書き換えるだけでなく、ドメイン全体の役割を見渡して、切替と切戻しを安全に設計する力が問われます。

このページの落ち

DNSは地味ですが、画面が開く前に必ず働いています。名前が間違えば、どれだけサーバーが元気でも誰もたどり着けません。
