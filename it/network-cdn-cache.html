Focus 11 / Network

CDN / Cache 近くの受付が、同じ内容をすばやく返します。

CDNは世界中のエッジでコンテンツを配信します。速くなる一方で、古い画面、キャッシュキー、TTL、パージが実務の焦点になります。

新メニューの画像を公開したのに、一部店舗では古い画像のままです。サーバー本体は更新済み。こんなとき、CDNやブラウザキャッシュが手前で応答しているかもしれません。

Quick Quiz 1

CDNで古い画面が残るときにまず見るものはどれでしょうか。

キャッシュの保持時間や指示を見ます。

Applied Quiz 2

オリジンログにリクエストが残っていません。利用者にはページが表示されています。疑いやすいものはどれでしょうか。

手前のキャッシュが返すと、オリジンにログが残らないことがあります。

このページの落ち

CDNは近道です。ただし近道に古い地図が置かれていると、みんな古い場所へ案内されます。

Network Specialist Written

CDN / Cache 午後記述

短答記述

別ユーザーの内容が表示されました。CDN設計で最も危険な点を答えてください。

事例ベースの午後問題

匿名ページは高速化したが会員ページで別人情報表示の疑いがあります。Cache-Control、Cookie、Authorization、キャッシュキー、パージで説明してください。

模範解答・採点観点

  • 個人別ページのキャッシュ、Cookie/Authorizationを含まないキーを挙げる。
  • private/no-store、Vary、除外設定、CDNログ、パージを確認する。
  • 高速化より情報漏えい防止を優先する。
