Focus 05 / Network

IP Address / Subnet Mask 住所を区切れば、ネットワークの形が見えます。

IPアドレスは端末の住所、サブネットマスクは町内の区切りです。CIDR、ネットワーク部、ホスト部、ブロードキャスト、設計まで厚く見ます。

カフェ本店、2号店、倉庫、本社をVPNでつなぐとします。全部を 192.168.1.0/24 にしてしまうと、どの店舗へ送ればよいか分からなくなります。IP設計は最初に雑にすると、あとで効いてきます。

CIDRを実務の感覚に変換します

サブネットは、端末を何台置けるかだけではなく、障害調査と拡張計画の単位でもあります。店舗のレジ、従業員Wi-Fi、監視カメラ、来客Wi-Fiを全部同じ範囲に入れると、あとで「どこが混んでいるのか」「どこを止めてもよいのか」が見えにくくなります。

CIDRよく見る用途使えるホスト数の目安設計メモ
/24店舗LAN、部署LAN254扱いやすい標準サイズ。雑に増やすとルートが増えます。
/25小さめの端末群126/24を半分に分けたいとき。
/26レジ、カメラ、AP管理62用途別に分けやすいサイズ。
/27サーバ管理、VPN機器30余裕は少ないので将来台数を確認します。
/30ルータ間接続2古典的なポイントツーポイント用。最近は/31も使われます。

現場で起きる事故

一番怖いのは重複です。本社と店舗が同じ 192.168.1.0/24 を使っていると、VPNでつないだ瞬間に宛先があいまいになります。次に怖いのは、最初に小さく切りすぎることです。監視カメラやタブレットが増えたとき、アドレスが足りなくなって休日作業で再設計になります。

Quick Quiz 1

192.168.10.23/24 の /24 が示すものはどれでしょうか。

/24 はCIDR表記です。255.255.255.0 と同じ範囲を表すことが多いです。

Applied Quiz 2

拠点間VPNで両拠点が 192.168.1.0/24 を使っています。起きやすい問題はどれでしょうか。

アドレス重複はVPN設計でよくある痛い問題です。
