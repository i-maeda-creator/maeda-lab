Focus 13 / Network

Mail / SMTP メールは、DNSと認証が絡む配送システムです。

SMTP、IMAP、MX、SPF、DKIM、DMARCを整理します。届かない、迷惑メールに入る、なりすまされる問題に効きます。

カフェが予約確認メールを送ります。本文を書くより前に、どのメールサーバーへ配送するか、送信元が本物か、迷惑メール扱いされないかが問題になります。

Mail / SMTP 午後記述

短答記述

SaaSから送るメールだけ迷惑メール扱いです。DNS認証で確認すべきレコードを答えてください。

事例ベースの午後問題

社内メール、MAツール、請求SaaSから送信します。請求SaaSだけSPF fail、DKIMも一部失敗します。SPF、DKIM、DMARC、Return-Pathで説明してください。

回答はこちらをクリック

模範解答・採点観点

  • SPF include、DKIM署名ドメイン、DMARC alignment、Return-Pathを挙げる。
  • DNS TXT、SaaS指定値、送信元IP、Authentication-Resultsを見る。
  • 本文ではなく配送認証とドメイン整合性を見る。

このページの落ち

メールは古いようで、DNSと認証のかたまりです。届かない理由は本文より配送経路にあります。
