Network

ネットワークは、データを別の場所へ届けるための階層的な仕組み。まずOSI参照モデルで役割を分け、実際のインターネットではTCP/IPとして理解する。

OSI参照モデル

OSI参照モデルは、ネットワーク通信を7つの層に分けて考えるための整理方法です。実際の通信機器やソフトウェアが完全にこの通りに分かれているわけではありませんが、「どの層が何を担当しているか」を考える地図になります。

  • L1 Physical: 電気信号、光、無線、ケーブル、コネクタ。
  • L2 Data Link: 同じネットワーク内でフレームを届ける。
  • L3 Network: IPアドレスを使って別ネットワークへ届ける。
  • L4 Transport: TCP/UDPでアプリ間の通信を扱う。
  • L5-L7 Session / Presentation / Application: アプリケーションに近い通信のルール。

TCP/IP

TCP/IPは、現在のインターネットで実際に使われている通信の考え方です。OSI参照モデルほど細かく7層に分けず、リンク層、インターネット層、トランスポート層、アプリケーション層として見ることが多いです。

OSIとTCP/IPの関係

OSIは学習や設計のための分類。TCP/IPは実際のインターネットで使う実装寄りの分類。まずOSIで役割を理解し、その後TCP/IPで現実の通信に接続するとわかりやすいです。

L1 Physical Layer

L1は、データを「物理的な信号」として運ぶ層です。ここではIPアドレスやHTTPの意味はまだ出てきません。0と1を、電圧、光、電波などに変えて相手まで届けます。

  • Cable: LANケーブル、光ファイバー。
  • Signal: 電気信号、光信号、無線信号。
  • Connector: RJ-45、SFPなど。
  • Bit: 0と1の最小単位。
  • Bandwidth: どれくらいの量を送れるか。

L1で見るポイント

L1の問題は「そもそも信号が届いているか」です。ケーブルが抜けている、規格が合っていない、距離が長すぎる、ノイズが多い、といった問題は上位層ではなくL1で考えます。
