01-5 / Network

L5-7 Application Side 人が見る通信は、だいたいここで起きています。

L5、L6、L7は、現代の学習ではまとめてアプリ側の通信として見ると分かりやすいです。 DNS、HTTP、TLS、API、認証、プロキシ、ログを通して、実際のWeb通信を追います。

カフェのスタッフがブラウザで注文管理画面を開きます。 画面にはただURLを打つだけに見えますが、裏側では名前解決、TLS、HTTPリクエスト、認証、API、ログが連続して動いています。

L5-7では、通信が人間に見える形へ近づきます。 「つながらない」だけでなく、「ログインできない」「証明書エラーが出る」「APIだけ500になる」のような話が増えます。

L5-7は、現代ではまとめて見ると使いやすいです

OSIではL5がセッション、L6がプレゼンテーション、L7がアプリケーションです。 ただし実務では、HTTP、TLS、DNS、API、認証、Cookie、プロキシのような具体的な仕組みで考えることが多いです。

Quick Quiz 1

ドメイン名をIPアドレスに対応させる仕組みはどれでしょうか。

正解はDNSです。URLを開く前に、まず名前からIPアドレスを調べる必要があります。

HTTPは、要求と応答の会話です

ブラウザはサーバーへリクエストを送り、サーバーはレスポンスを返します。 200、301、403、404、500のようなステータスコードは、アプリ側の切り分けでとても役立ちます。

コード意味現場での見方
200成功アプリが応答している
301/302リダイレクトURL転送、HTTPからHTTPSへの転送
403禁止権限、WAF、アクセス制限
404見つからないURL、ルーティング、公開設定
500サーバー内部エラーアプリ、DB、設定、例外ログ

Quick Quiz 2

HTTPステータスコード404の意味として近いものはどれでしょうか。

正解は「見つからない」です。URLやルーティング、公開設定の確認につながります。

TLSは、HTTPSの安心感を支えます

TLSは通信を暗号化し、証明書によって相手の正しさを確認します。 証明書期限切れ、CN/SAN不一致、中間証明書不足、古いTLSバージョンは、実務でよく出るトラブルです。

Quick Quiz 3

HTTPSで、通信の暗号化や証明書確認に関係が深いものはどれでしょうか。

正解はTLSです。HTTPSはHTTPをTLSで保護している、と考えると整理しやすいです。

Field Quiz A

証明書の期限切れで起きやすい現象はどれでしょうか。

正解です。証明書の期限や名前の不一致は、HTTPSでよくある実務トラブルです。

Field Quiz B

APIだけ500エラーになるとき、まず見たいものはどれでしょうか。

正解です。500はアプリ側の内部エラーを示すことが多いので、ログと依存先を確認します。

応用です。アプリ側のトラブルを切り分けます

Applied Quiz 6

IPアドレスを直接指定するとつながりますが、ドメイン名ではつながりません。まず疑いやすいものはどれでしょうか。

正解です。IPで届くなら、次は名前解決を疑います。

Applied Quiz 7

ログイン後だけ403になります。L7としてまず見たいものはどれでしょうか。

正解です。ログイン後だけなら、認証よりも認可やセッション、WAFの判定が関係することがあります。

Applied Quiz 8

HTTPS化した直後、一部ブラウザだけ証明書エラーになります。疑いやすいものはどれでしょうか。

正解です。一部環境だけ失敗するTLS問題は、中間証明書や対応バージョンの差が効くことがあります。

最後に、IPAの試験っぽくまとめて確認しましょう

IPA Style Test 1

HTTPステータスコード500の説明として近いものはどれでしょうか。

正解はサーバー内部エラーです。アプリログや依存先を確認します。

IPA Style Test 2

HTTPSでサーバー証明書の検証に失敗したとき、利用者側で起きやすいことはどれでしょうか。

正解です。証明書の期限、名前、信頼チェーンはHTTPSの重要ポイントです。

ネットワークスペシャリスト級チャレンジです

Network Specialist Challenge 1

CDN導入後、一部地域だけ古い画面が表示されます。サーバー本体のログには新しいリクエストが来ていません。疑いやすいものはどれでしょうか。

正解です。CDNが応答している場合、オリジンサーバーにログが残らないことがあります。キャッシュの設計を見ます。

Network Specialist Challenge 2

API Gateway経由のAPIだけ401になり、直接アプリへ送ると200になります。最も疑いやすいものはどれでしょうか。

正解です。401は認証まわりです。Gatewayを通したときだけなら、トークン検証やヘッダーの渡し方を確認します。

このページの落ち

L5-7は、人が「Webが変」「ログインできない」「APIが落ちた」と感じる場所です。 でも中身は、DNS、TLS、HTTP、認証、ログの小さな積み重ねです。 アプリ側の通信は、画面の裏でかなり几帳面に働いています。
